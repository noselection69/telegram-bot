🚀 РЕКОМЕНДУЕМЫЙ ПУТЬ (15-20 МИНУТ):

  ВАРИАНТ 1: Railway.app ⭐ ЛУЧШЕ ВСЕГО

    Шаг 1️⃣: Подготовка (5 минут)
      ├─ git init
      ├─ git add .
      ├─ git commit -m "Deploy ready"
      ├─ git remote add origin https://github.com/YOUR_USERNAME/telegram-bot.git
      └─ git push -u origin main

    Шаг 2️⃣: GitHub (1 минута)
      └─ Проверить что код есть на GitHub (PUBLIC!)

    Шаг 3️⃣: Railway (5 минут)
      ├─ https://railway.app
      ├─ "Start Project" → "Deploy from GitHub"
      ├─ Авторизуйся и выбери репо
      └─ Добавь переменные (BOT_TOKEN, WEBHOOK_URL)

    Шаг 4️⃣: Проверка (3 минуты)
      ├─ Посмотри логи на Railway
      ├─ Проверь /start в Telegram
      ├─ Откройте Web App
      └─ Протестируй функции

    ✅ ГОТОВО! Бот работает 24/7!


  ВАРИАНТ 2: Render.com (если Railway не нравится)
    ├─ Похож на Railway
    ├─ Чуть более сложный
    └─ Более стабильный на платных планах

  ВАРИАНТ 3: VPS (для опытных)
    ├─ Полный контроль
    ├─ Требует знаний Linux
    └─ Дешево (50-150₽/месяц)

🔧 ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ (нужно добавить на хостинге):

  Обязательные:
    BOT_TOKEN = ваш_токен_бота (получи в BotFather)
    WEBHOOK_URL = https://app-name.railway.app

  Опциональные:
    FLASK_ENV = production
    DEBUG = False
    DATABASE_URL = postgresql://... (если используешь PostgreSQL)
